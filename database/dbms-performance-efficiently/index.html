DBMS 성능 측정 방법

1. 고가용성을 위한, DBMS 성능 측정 방법 개요​

개념

DBMS의 성능 측정을 위해 벤치마크 및 시나리오 기반의 테스트등을 수행하여 성능 특성을 파악하는 활동

ISO/IEC 25010

품질특성을 평가를 위한 모델 ISO/IEC 25010의 실행 효율성 항목을 활용하여 "시간효율성", "자원활용성", "기억용량" 등을 측정

ㅡ DBMS 성능 지표를 설정하여 도메인 분석 후 평가 수행

2. DBMS 성능 측정을 위한 성능 지표 설명​

항목측정지표설명
시간
효율성		응답시간(초)- Requst time + Resposne time
처리량(TPS)- 시간당 Transaction 처리
반환시간(초)- 시스템 Transaction 실행부터 사용자 반환 시간
자원
활용성		CPU, Memory (%)- 수행 시 CPU, Memory 의 사용량 모니터
tmpC- 시간처리량 / 자원사용
기억용량파일 I/O- 데이터 파일 또는 로그 파일에 기록되는 용량
인덱스- 구성에 필요한 데이터의 단위
페이지
기대수명		버퍼캐시 또는 페이지 메모리의 저장 가능 용량

ㅡ 측정 지표를 바탕으로 베치마킹, 부하, 실험적 테스트 기법으로 평가

3. DBMS 성능 측정 방법 설명​

가. DBMS 성능 측정 방법의 도식화​

나. DBMS 성능 측정 방법 상세 설명​

구분평가방법설명
벤치
마크
평가		TPC-C- OLTP 벤치마크
- 트랜젝션 중심
TPC-H- OLAP 벤치마크
- 복잡한 SQL 쿼리
TPC-DS- 데이터웨어하우스
- BI
TPC-E- OLTP (금융 관련 트랜젝션 중심)
TPC-VMS- 가상머신에서 성능 측정
- 효율성 및 오버해드 분석
YCSB- NoSQL 벤치마크
- 비정형 데이터 측정
부하
Test		정적부하- 일정한 부하를 지속적으로 주입
동적부하- 사용량을 동적으로 변화시켜 주입
스트레스- 시스템의 한계 지점까지 부하를 주입
실험적
Test		샘플데이터- 샘플 데이터를 활용한 유사 환경의 테스트
실제데이터운영환경과 유사한 실제 데이터 기반 테스트
시뮬레이션특정 시나리오 기반 기능 수행 테스트

ㅡ DBMS 성능 측정은 도메인 최적화 설계 등의 고려 필요

IV. DBMS 성능 측정 시, 고려 사항 설명​

(1) 도메인 최적화 설계(2) 벤더 종속성 배제
(3) Test 환경 구성 최적화(4) Application 환경 구성 고려
